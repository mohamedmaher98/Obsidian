   # Project DSL Conventions

When generating DSL entity classes in this project, always follow these conventions:

1. **Entity Definition**
   - Use `@NaMaEntity` annotation for each entity class.
   - If the entity is a master file, it must extend `MasterFile`.

2. **Fields**
   - Use `@NaMaField(required = true)` for mandatory fields.
   - Use `@NaMaField(hasEmptyPrefix = true)` when the field requires an empty prefix.
   - Always add `@TranslateField(en = "English Name", ar = "Arabic Name")` for fields shown to the user.
   - If a translation entry does not exist in `cmn-gen`, create it automatically.

3. **Contact Information**
   - Contact fields (`responsibleName`, `phoneNumber`, `email`) must always be grouped inside a separate `ContactInfo` class.
   - Entities should reference `ContactInfo` instead of repeating these fields.

4. **Data Types**
   - Always use DSL data types (`TextDF`, `PhoneNumber`, `Currency`, `BooleanDF`, `LongTextDF`, `LargeData`, etc.).
   - Avoid plain Java types (`String`, `int`, etc.).

5. **Naming**
   - Use camelCase for all field names.
   - Names must be descriptive and meaningful.
