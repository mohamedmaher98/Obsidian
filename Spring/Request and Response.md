الـ **Request** و **Response** في **API** هما أساس تفاعل العميل (Client) مع الخادم (Server). في التطبيقات الحديثة مثل الـ **RESTful APIs**
تم استخدام هذين المفهومين لتمرير البيانات بين الخادم والعميل
### ==1-Request== 

1- HTTP Request 
الـ **HTTP Request** هو طلب يتم إرساله من العميل إلى الخادم للحصول على بيانات أو تنفيذ عملية معينة على الخادم.
يتكون الـ **Request** من عدة أجزاء:
**الـ HTTP Method (الطريقة)**:

- تُستخدم لتحديد نوع العملية التي يجب على الخادم القيام بها.
    
- **الأمثلة**:
    
    - `GET`: لاستخراج البيانات من الخادم.
        
    - `POST`: لإرسال بيانات إلى الخادم.
        
    - `PUT`: لتحديث البيانات.
        
    - `DELETE`: لحذف البيانات.
---
2- **الـ URL (رابط الـ API)**:


- يُستخدم لتحديد المورد (Resource) الذي يتم الوصول إليه.
- ex:/users/123
- 
3-  **الـ Headers (الرؤوس)**:

- تتضمن معلومات إضافية حول الطلب مثل نوع البيانات (Content-Type)، وعنوان الـ API Key، وإعدادات التوثيق.
  
  4- ال body
- **الـ Body (الجسم)**:

- يحتوي على البيانات المرسلة في بعض الـ Requests (مثل `POST` أو `PUT`). يختلف التنسيق حسب الـ API (غالبًا يكون `JSON` أو `XML`).
ex : 
- {
    "name": "John Doe",
    "email": "john@example.com"
}
---
### ==2-Response== 
HTTP Response (استجابة HTTP)
   الـ **HTTP Response** هو رد من الخادم إلى العميل بعد تلقي الطلب. يشمل معلومات حول النتيجة سواء كانت ناجحة أو فاشلة.
   #### **أجزاء الـ HTTP Response**:

**الـ Status Code (رمز الحالة)**:
    
    - يُظهر ما إذا كان الطلب ناجحًا أم فاشلاً.
        
    - **الأمثلة**:
        
        - `200 OK`: الطلب ناجح وتم معالجة البيانات.
            
        - `201 Created`: تم إنشاء المورد بنجاح (غالبًا عند استخدام `POST`).
            
        - `400 Bad Request`: الطلب يحتوي على أخطاء.
            
        - `404 Not Found`: المورد غير موجود.
            
        - `500 Internal Server Error`: خطأ في الخادم.
